Deploying Autopilot Enabled Machines

Prerequisites before Autopilot Device Setup

  1. Have the user or manager of new hire fill out the Device Setup Ticket form on the H2M Helpdesk.

Device Configuration (Autopilot Stager)

  1. When prompted with the H2M logoed login screen, login with the "Autopilot Stager" account.
  2. Rename the device to fit the naming scheme, (OFFICE CODE-username), then restart.
  3. After computer name is changed, open Active Directory. Navigate to the _Autopilot Creation OU. Move the computer object to the proper OU so GPOs can be applied to the machine.
  4. Add device to "Intune-Setup-Devices" group in Entra.
  5. Complete the approriate tasks in the checklist

Device Configuration (End User)

  1. Set the assigned user as the primary user in Intune, then sync in Company Portal.
  2. For users in the same office, have the user come to the IT room to log into the laptop before deploying it.
    • For remote users, ship the laptop to user/office, remote into laptop after it arrives and then continue the process from there. Once the user confirms everything is on the new laptop, send old laptop back to Melville
  3. Complete the rest of the checlist

Deploying Non-Autopilot Enabled Spare Machines

Setup on Device

  1. Wipe the machine using the latest version of Windows OS.
  2. After successful Windows installation, the machine will display an option to name the device.
    • Name the device with the proper naming scheme, (OFFICE CODE-spare), then click next.
  3. Choose to setup the device for work or school.
  4. Choose to domain join the device, then when prompted to set things up for work or school, choose sign in options.
  5. Create the local "h2m" account with the standard password and disable all privacy settings.
  6. After logging into the local "h2m" account, the machine will display the desktop. The first thing that should be done is to confirm Windows is activated. If this option does not appear, then move to the next step.
  7. The next step is to join the device to the domain.
    • Go to Settings, choose the System tab, scroll down to About, choose Advanced System Settings, choose "Computer Name" tab and change the domain to "h2m.com". Then restart.
  8. When the device is connected to the Domain, log in as the "Stager" user.
  9. After logging in as the user, open Active Directory. Navigate to the Computers OU. Move the computer object to its respective OU so GPOs can start to be applied to the machine.
  10. After the device is moved to the OU, open command prompt as the user and run "gpupdate /force".
  11. To establish a connection to Microsoft Intune, open the "Task Scheduler" and navigate to "Task Scheduler Library > Microsoft > Windows > Workplace Join"
    • Right click the "Device-Sync" task and run it. This will trigger a link to Microsoft Intune.
  12. After the device has been successfully enrolled in Intune (confirmed by the presence of the Intune Device Extension in the Control Panel), please proceed with and complete the checklist.
